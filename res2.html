Compte Rendu - pfSense & Proxmox

Objectifs

1. Objectifs

Le but de cette activité est de créer un réseau de virtualisation avec pfSense afin de gérer le routage, la sécurité et l'accès au réseau.

Problématique

2. Problématique

Comment garantir un routage en virtualisation ? Comment sécuriser les échanges réseau et éviter les accès non autorisés ?

Risques

3. Sécurité & Impacts

Risque d'accès non autorisé entre les réseaux. Une mauvaise configuration peut également empêcher l'accès à Internet et bloquer le travail des utilisateurs.

Matériel

4. Présentation du matériel

2 machines utilisateurs, un Mini-PC, pfSense, un routeur, un switch, Rufus pour créer la clé USB bootable et Draw.io pour réaliser les schémas.

Câblage

5. Schéma de câblage

Brancher le Mini-PC au réseau, connecter le routeur au switch puis raccorder le switch à pfSense.

Draw.io

6. Schéma pfSense

Réalisation du schéma du circuit réseau sous Draw.io afin de représenter l'architecture et les connexions.

Proxmox

7. Configuration du Mini-PC

Télécharger Proxmox VE et Rufus. Configurer le proxy puis attribuer une adresse IP pour accéder à l'interface.

Commandes

8. Commandes utiles

ip addr : afficher les paramètres réseau. nano /etc/network/interfaces : modifier la configuration réseau.

Adressage

9. Configuration IP

Utilisation de Set Interface IP Address. Attribution de l'adresse 192.168.2.2 avec le masque 255.255.255.0.

Installation

10. Installation de Proxmox

Démarrer le Mini-PC avec la touche Suppr ou F7. Vérifier le port COM utilisé depuis le Gestionnaire de périphériques.

Pare-feu

11. Désactivation du Pare-feu

Désactiver le Pare-feu Windows Defender afin d'éviter les blocages de communication réseau durant les tests.

Configuration

12. Paramètres Réseau

Configurer l'adresse IP, le masque de sous-réseau, la passerelle et le DNS préféré depuis les paramètres Ethernet.
